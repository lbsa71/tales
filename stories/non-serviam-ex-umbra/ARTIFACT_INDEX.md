## Conventions

- **Real artifacts** carry full bibliographic citation: title, author, publication, date, URL where applicable.
- **Invented artifacts** carry in-universe attribution: a fictional source name, date, and (where relevant) a noted real-world model the invention is calibrated against.
- **Sensitivity**: real worker testimony is quoted only with verified permission status (or under fair use as published material). The status is recorded here.
- **Speakability**: every artifact is confirmed readable aloud. The "Voice notes" column captures any reading-aloud notes (e.g., "read as quoted dialogue, not as screen capture").

## Reading-aloud verification

Each artifact is run through this test:

1. Read the artifact aloud (without the prose around it).
2. Note any place a reader has to look at formatting to know how to voice it.
3. Each formatting-dependent moment is a defect; rewrite or restructure.

This is most acute for:
- Tables in real artifacts (often need reformatting as sequential lists)
- Email/Slack threads (need clear "X wrote: ..., Y wrote: ..." voicing)
- Phone-tree transcripts (need clear "Press 1 for X, press 2 for Y, ..." voicing)
- Code commit messages (read as quoted technical-message; line-breaks lost)
- System log entries (read in full, with brief intro: "Log line 4471 read: ...")
