# Audio and Visual Module Mapping

## Complete Module Alignment

This document provides a comprehensive mapping between visual and audio modules for each evolutionary rung in the Unsung Heroes interactive reader.

## Overview

Each of the 9 chapters (evolutionary rungs) has both a visual and audio module that work together to create a cohesive multi-sensory experience. The modules are carefully designed to complement each other without creating sensory overload or distracting from the text.

## Detailed Rung Breakdown

### Rung 1: Pre-cellular Replicators
**Theme**: Chemical persistence

**Visual Module**: DLA (Diffusion-Limited Aggregation)
- Slow particle wandering
- Organic branching patterns
- Warm gold color (#c89b5c)
- 30-50 particles, sparse movement

**Audio Module**: Filtered Noise
- Continuous pink noise with modulated filter
- Ultra-rare soft grain events (< 0.1% density)
- Frequency range: 100-600 Hz
- Character: Cosmic hum, pre-semantic soundbed

**Combined Experience**: Visual shows particles slowly aggregating while audio creates a diffuse, primordial atmosphere. Together they evoke the emergence of pattern from chaos.

## Design Principles for Combined Experience

### Complementary, Not Competing

1. **Visual Leads**: Visual modules are slightly more prominent (opacity 0.3-0.5)
2. **Audio Supports**: Audio modules stay very quiet (gain 0.14-0.20)
3. **No Sensory Overload**: Combined effect remains meditative and contemplative

### Synchronized Transitions

1. **Timing**: Visual fades (1200ms), Audio crossfades (3000ms)
2. **Coordination**: Both systems transition on same trigger
3. **Smooth Blend**: Longer audio crossfade prevents jarring changes

### Thematic Consistency

1. **Color/Timbre**: Warm visuals pair with warm timbres
2. **Density/Density**: Sparse visuals pair with sparse audio events
3. **Movement/Movement**: Visual motion echoed in audio spatial movement

### Performance Balance

1. **CPU Budget**: Visual ~10-15%, Audio ~5%, Total <25%
2. **Memory Budget**: Visual ~50MB, Audio ~10MB, Total <100MB
3. **Independence**: Each system can run without the other

## User Control Strategy

### Global Controls

- **Enable Visuals**: Toggle all visual modules on/off
- **Enable Audio**: Toggle all audio modules on/off
- **Audio Volume**: Global volume slider (0-100%)
- **Quick Mute**: Instant audio mute button

### Accessibility

- **Prefers-Reduced-Motion**: Reduces both visual and audio density
- **Visual-Only Mode**: Audio disabled completely
- **Audio-Only Mode**: Visuals disabled completely (rare but possible)

### Smart Defaults

- **Desktop**: Both visual and audio enabled at default volumes
- **Mobile**: Visual enabled, audio disabled by default (battery/data)
- **Low-Power**: Both systems reduce complexity automatically

## Implementation Notes

### Shared Resources

Both systems share:
- React context for current chapter
- Transition timing triggers
- Performance monitoring
- User preference storage

### Independent Operation

Each system maintains:
- Own lifecycle management (Orchestrator)
- Own configuration (visualConfig.json, audioConfig.json)
- Own module implementations
- Own cleanup procedures

### Coordinated Transitions

```typescript
// In Reader.tsx
const handleChapterTransition = async (newIndex: number) => {
  // Both systems receive same trigger, handle independently
  await Promise.all([
    visualOrchestrator.transitionTo(newIndex),
    audioOrchestrator.transitionTo(newIndex)
  ]);
};
```
